For machine learning, I study the textbook “Machine Learning with PyTorch and Scikit-learn book -- Sebastian”. From the book, we can define machine learning steps as followings:

  • Data exploration, data cleaning, feature engineering & dimensionality reduction
  • splitting dataset into train data and test data
  • train data combine with algorithms
  • generating train model
  • model scoring
  • re-train model
  • model evaluation
  • model prediction & model deployment

They are explained as follows:

  • Data exploration, data cleaning, feature engineering & dimensionality reduction:
    • Data exploration & data cleaning: (focus on the data content)
      • Visualization data: Use bar charts, histograms, kernel density plot, seaborn, scatter plot, deal with overplotting, desity-2d plot, box plot, violin plot to know the relationship between features and to figure out the possible problem in data.
      • Data cleaning: Data cleaning for the missing values and duplicated rows. Transform the column data type. Process the imbalanced data, recode the column names, etc.
    • Feature engineering: (focus on the data features)
      • Transforming numeric variables, compute new features, aggregating categories variables.
      • Use one hot encoding to transform categorical features (ordinal features or nominal features).
      • Feature scaling (normalization or standardization), transform date code, regularization (L1, L2), etc.
    • Dimensionality reduction: The purpose is to reduce the complexity of the model and avoid overfitting. From the other viewpoint, the data size is reduced but the data information is same. The methods are feature selection and feature extraction which explain as following:
      • Feature selection: Two ways to get features importance, SBS and Random Forest.
      • Feature extraction: There are three ways, PCA (Principal Component Analysis), LDA (Linear Discriminant Analysis), Kernel PCA.
  • model selection:
    • The six main functions of scikit-learn are classification, regression, clustering, dimensionality reduction, model selection, preprocessing.
    • Classification algorithm:
      • KNN, Logistic Regression, Support Vector Machine, Decision Tree, and Random Forest. We can choose the proper algorithm solve certain problem. Or if you don’t how to choose, pipeline or ensemble learning together with grid search CV method can help you to find the “best” algorithm.
  • Performance tuning:
    • Using grid search CV to find out optimal hyperparameters combinations. (Spending money to buy time.)
    • Using pipeline to simplify code and easy to tune hyperparameters. To find the best performance.
    • K-fold cross-validation/Holdout cross validation: to avoid the sample data to influence model performance.
    • Regularization L1 & L2: to solve overfitting problems.
    • Overall speaking, using pipeline is for tuning hyperparameter. After tuning hyperparameter, we want to make use of k-fold to avoid possibility of sampling data influencing model result. Another factor also affects model performance is imbalance data. For this case, something we can do:
      • Try to get more data.
      • Change performance index, for example, accuracy or loss is not so important to imbalanced data. We can change to use confusion matrix index, like precision, recall. (so-called f1-score).
      • SMOTE: Dropping certain proportion of the majority data and increasing the minority data size by using normal distribution, something like that random forest. Using SMOTE to make the data more balanced.
  • Performance index:
    • Confusion metrics: The indicators in common use are: Accuracy, Precision, Recall, F1 score, sensitivity.
    • AUC/ROC:
      • ROC (Receiver Operator Characteristic): Based on each decision threshold, to compare the change between true positive rate and false positive rate.
      • AUC (Area Under the Curve): the larger area of AUC is the better performance of model.
  • Ensembling Learning: We can use many classification methods by using ensemble classifier. Four methods of ensemble learning:
    • Majority Voting: the majority voting can win the prediction.
    • Bagging: Resample with replacement to generate enough bootstrap samples. Increase new data to enhance and modify the bootstrap data by using normal distribution.
    • Boosting: Focus on hard classified sample and learn from error judgement. Amplify the weak part during learning process and expect to get better performance.
    • Adaboost (Adaptive Boosting): Use full sample for training and make bigger weighting for error judgement sample. Retrain the model.
